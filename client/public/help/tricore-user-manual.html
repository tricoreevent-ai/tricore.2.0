Platform Overview

What TriCore Events does

TriCore Events is a sports event management system. It helps the team publish events, collect registrations, review payments, manage schedules, record finance, and control the whole website from one admin portal.

Public users

Find events, unlock details with Google sign-in, register, pay, and track their own status.

Internal staff

Create events, run match operations, manage audience communication, track finance, and control access.

System purpose

Keep event operations, participant communication, and business records in one place.

Registration Process

How a participant registers

Step 1 Open the website and click Events.
Step 2 Choose an event, then continue with Google sign-in to unlock the full page.
Step 3 Review event details, match schedule, and status. Fill the registration form when it is open.
Step 4 Submit the form. Free events save directly. Paid events move to payment.
Coming Soon events: when an event is not open yet, users see Notify Later instead of the registration form. Their contact details are saved for future follow-up. After sign-in, the user dashboard also shows registrations, upcoming matches, and refund / payout details.
Payment Process

How payment works for paid events

Available methods

QR, UPI, or bank transfer, depending on what the admin has enabled in settings.

What the user submits

Payment amount, reference number if available, and screenshot or proof.

Possible statuses

Pending, Under Review, Confirmed, or Failed after admin verification.

Admin Actions

What the admin does after registration and payment

  1. Open Registrations from the admin portal.
  2. Select the date range and click Apply Filters.
  3. Open the registration record and review the payment status.
  4. Check proof, amount, reference, and event details.
  5. Confirm, follow up, or reject based on the case.
  6. Export the list if finance or operations needs an offline file.
Accounting Guide

Detailed guide to Accounting

The Accounting page has three main work areas: Record Transaction, Transaction Ledger, and Manage Categories. Reports now live in the main Reports menu, and transaction edits can require OTP approval when security settings enable it.

Record Transaction

Core fields

Transaction type, date, event or common ledger, category, amount, payment method, and reference.

Supporting details

Document number, payment reference, voucher number, and notes for clarity.

Invoice option

Turn on invoice generation only when a sponsor or vendor needs a formal document.

Transaction Ledger

The ledger loads only after filters are applied. Use the date range first, then add event, type, scope, category, payment mode, or source as needed, and click Apply Filters.

Manage Categories

This section keeps the financial category list clean. Admins can add, edit, and delete categories so reports stay consistent and duplicate labels are avoided.

1. RecordEnter the main money movement first.
2. ExplainAdd notes or supporting reference details.
3. InvoiceOpen billing fields only if a document is needed.
4. SaveSave the entry and print or export later if needed.

Troubleshooting

Common problems and quick checks

Transaction missing

Check the date range, filters, and whether the transaction was saved successfully.

Payment still pending

Check if proof was uploaded, the amount matches, and the reference number is clear.

Event access locked

Check that the user signed in with Google and that the event is still open or coming soon.

Report is empty

Check the selected date range and confirm records actually exist for that period.

Backup tools failing

Use Settings → Backups → Get Database Info to confirm the live MongoDB connection and size details.

Campaign email not sending

Check SMTP readiness, test email delivery, approval status, and unsubscribe suppression.

Best Practices

Daily habits for smooth operations

  • Always check the date range before saying data is missing.
  • Use clear references when recording transactions.
  • Confirm payments before generating schedules.
  • Review AI fixture suggestions before approving the final schedule.
  • Keep accounting categories clean and avoid duplicates.
  • Use exports for offline records and sharing.
  • Review alerts from Overview or Reports every day.
  • Review audience opt-outs before launching campaigns.
  • Use the built-in User Manual during onboarding and refresher training.
  • Check backup database info before assuming a restore file is the problem.
